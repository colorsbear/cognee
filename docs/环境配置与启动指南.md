## 一、快速启动（最小配置）

### 1.1 环境要求

- **Python**: 3.10 - 3.12（推荐 3.11）
- **包管理器**: uv（推荐）或 pip 或 poetry

### 1.2 安装步骤

```bash
# 1. 创建虚拟环境（使用 uv，更快）
uv venv
# 或使用 Python venv
python -m venv .venv

# 2. 激活虚拟环境
# Windows PowerShell/CMD:
.venv\Scripts\activate
# Linux/Mac:
source .venv/bin/activate

# 3. 安装 Cognee（最小安装）
uv pip install -e .
# 或使用 pip
pip install -e .

# 4. 安装开发依赖（可选）
uv pip install -e ".[dev]"
```

### 1.3 配置环境变量

复制模板文件：
```bash
# Windows
copy .env.template .env

# Linux/Mac
cp .env.template .env
```

**最小配置（仅使用 OpenAI）**:
```bash
# .env 文件
LLM_API_KEY="sk-your-openai-api-key"
LLM_MODEL="openai/gpt-4o-mini"
LLM_PROVIDER="openai"
```

**默认数据库**（无需额外配置）：
- 关系数据库：SQLite（自动创建）
- 向量数据库：LanceDB（自动创建）
- 图数据库：Kuzu（自动创建）

### 1.4 验证安装

```python
# test_cognee.py
import asyncio
import cognee

async def test():
    # 1. 添加数据
    await cognee.add("Cognee 是一个 AI 记忆平台", dataset_name="test")
    print("✓ 数据添加成功")
    
    # 2. 构建知识图谱
    await cognee.cognify(datasets=["test"])
    print("✓ 知识图谱构建成功")
    
    # 3. 搜索
    results = await cognee.search("Cognee 是什么？")
    print(f"✓ 搜索结果：{results}")

if __name__ == "__main__":
    asyncio.run(test())
```

运行测试：
```bash
python test_cognee.py
```

## 三、启动问题排查

### 3.1 常见问题速查表

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `LLM connection test timed out` | LLM 端点不可达 | 检查 API Key 和端点 URL |
| `DatabaseNotCreatedError` | 数据库未初始化 | 运行 `await cognee.add()` 初始化 |
| `Empty knowledge graph` | 未运行 cognify | 运行 `await cognee.cognify()` |
| `Permission Denied` | 多租户权限问题 | 检查用户数据集权限 |
| `No module named 'xxx'` | 缺少依赖 | 安装对应的 extras |

---

### 3.2 详细问题排查

#### 问题 1: LLM 连接超时

**错误信息**:
```
LLM connection test timed out after 30s. Check that your LLM endpoint is reachable and responding.
```

**原因分析**:
1. API Key 不正确或已过期
2. 端点 URL 配置错误
3. 网络防火墙阻止请求
4. LLM 服务暂时不可用

**排查步骤**:

```bash
# 1. 测试 API 端点连通性
# OpenAI
curl -X POST https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer $LLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"test"}]}'

# Azure OpenAI
curl -X POST $LLM_ENDPOINT \
  -H "api-key: $LLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"test"}]}'

# 阿里云 DashScope（注意：需要正确的 API Key 格式）
curl -X POST https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions \
  -H "Authorization: Bearer $LLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen3.5-plus","messages":[{"role":"user","content":"test"}]}'
```

**解决方案**:

1. **检查 .env 配置**:
```bash
# 确保没有多余的空格和引号
LLM_API_KEY="sk-bb02112a93104cae8a26e3314573df22"  # ✓ 正确
LLM_API_KEY=sk-bb02112a93104cae8a26e3314573df22       # ✓ 也正确
LLM_API_KEY=" sk-bb02112a93104cae8a26e3314573df22 "  # ✗ 错误，有空格
```

2. **跳过连接测试**（开发环境）:
```bash
COGNEE_SKIP_CONNECTION_TEST=true
```

3. **增加超时时间**:
```bash
# 在某些 LLM 较慢的情况下
LLM_TIMEOUT="60"
```

4. **使用代理**（如果在国内访问 OpenAI）:
```bash
# 设置 HTTP 代理
HTTP_PROXY="http://proxy-server:port"
HTTPS_PROXY="http://proxy-server:port"

# 或在代码中设置
import os
os.environ["HTTP_PROXY"] = "http://proxy-server:port"
os.environ["HTTPS_PROXY"] = "http://proxy-server:port"
```

#### 问题 2: 数据库未创建

**错误信息**:
```
DatabaseNotCreatedError: Search prerequisites not met: no database/default user found.
```

**原因**: 在调用 `search()` 之前没有先调用 `add()` 和 `cognify()`

**解决方案**:

```python
import asyncio
import cognee

async def fix():
    # 正确的顺序
    await cognee.add("Your data here", dataset_name="my_dataset")  # 1. 先添加数据
    await cognee.cognify(datasets=["my_dataset"])                   # 2. 构建图谱
    results = await cognee.search("Your query")                     # 3. 最后搜索
    
asyncio.run(fix())
```

**手动初始化数据库**:
```python
import cognee
await cognee.setup()  # 显式初始化数据库和表
```

#### 问题 3: 图谱为空

**错误信息**:
```
Search attempt on an empty knowledge graph
```

**日志信息**:
```
Dataset 'my_dataset' has X data item(s) but the knowledge graph is empty.
Please run cognify to process the data before searching.
```

**原因**: 数据已添加但未运行 `cognify()` 处理

**排查步骤**:

```python
# 1. 检查数据集是否有数据
from cognee.modules.data.methods import get_dataset_data
from cognee.infrastructure.databases.graph import get_graph_engine

# 2. 检查图谱是否为空
graph_engine = await get_graph_engine()
is_empty = await graph_engine.is_empty()
print(f"Graph empty: {is_empty}")

# 3. 运行 cognify
await cognee.cognify(datasets=["my_dataset"])
```

**完整流程示例**:
```python
import asyncio
import cognee

async def complete_flow():
    dataset_name = "test"
    
    # Step 1: Add data
    print("Adding data...")
    await cognee.add("""
        Cognee is an AI memory platform.
        It uses knowledge graphs for better reasoning.
        The main workflow is: add -> cognify -> search.
    """, dataset_name=dataset_name)
    
    # Step 2: Build knowledge graph
    print("Building knowledge graph...")
    await cognee.cognify(datasets=[dataset_name])
    
    # Step 3: Search
    print("Searching...")
    results = await cognee.search("What is Cognee?")
    print(f"Results: {results}")

asyncio.run(complete_flow())
```

#### 问题 4: 嵌入维度不匹配

**错误信息**:
```
ValueError: Vector dimension mismatch: expected 3072, got 1536
```

**原因**: 嵌入模型配置的维度与实际模型输出不一致

**解决方案**:

1. **确认嵌入模型维度**:
```python
# OpenAI 嵌入模型维度
# text-embedding-3-small: 1536
# text-embedding-3-large: 3072
# text-embedding-ada-002: 1536

# 修改 .env
EMBEDDING_MODEL="openai/text-embedding-3-small"
EMBEDDING_DIMENSIONS=1536  # 与模型匹配
```

2. **重建向量索引**:
```python
# 删除旧的向量数据库文件
import shutil
shutil.rmtree("./lancedb_data")

# 重新运行 cognify
await cognee.cognify(datasets=["my_dataset"])
```

#### 问题 5: 多租户权限问题

**错误信息**:
```
Permission Denied: User does not have 'read' permission on dataset
```

**搜索返回空列表但无错误**（这是安全特性，防止信息泄露）

**原因**: 多租户访问控制启用，用户没有数据集权限

**解决方案**:

1. **检查是否启用了多租户**:
```bash
# .env
ENABLE_BACKEND_ACCESS_CONTROL=true  # 默认启用
```

2. **使用正确的用户**:
```python
from cognee.modules.users.methods import get_user_by_email

# 获取有权限的用户
user = await get_user_by_email("user@example.com")
results = await cognee.search("query", user=user, datasets=["my_dataset"])
```

3. **访问旧数据**（如果是从旧版本升级）:
```bash
# 临时禁用多租户访问旧数据
ENABLE_BACKEND_ACCESS_CONTROL=false
```

4. **授予权限**:
```python
from cognee.modules.users.methods import grant_dataset_permission

await grant_dataset_permission(
    user_id=user.id,
    dataset_id=dataset.id,
    permission_type="read"
)
```

#### 问题 8: 内存不足

**错误信息**:
```
MemoryError: Unable to allocate X GiB
```

**原因**: 处理大型文档时内存不足

**解决方案**:

1. **减小批量大小**:
```python
await cognee.cognify(
    datasets=["large_dataset"],
    chunks_per_batch=10,  # 默认 100，减小以减少内存使用
    data_per_batch=5      # 默认 20
)
```

2. **使用流式处理**:
```python
# 分批次添加数据，而不是一次性加载所有
for chunk in large_data_chunks:
    await cognee.add(chunk, dataset_name="large_dataset")
```

3. **增加系统虚拟内存**（Windows）:
```
系统属性 → 高级 → 性能 → 高级 → 虚拟内存 → 增加页面文件大小
```

```yaml

  cognee:
    image: cognee/cognee:latest
```

```python
import shutil
import os

def cleanup():
    """清理 Cognee 数据以便重新开始"""
    paths_to_remove = [
        "./cognee/.cognee_system",
```

## 十、总结

### 快速故障诊断流程

```
1. 检查 .env 配置是否正确
   └─> LLM_API_KEY, LLM_MODEL, LLM_PROVIDER

2. 测试 LLM 连接
   └─> curl 测试 API 端点
   └─> 设置 COGNEE_SKIP_CONNECTION_TEST=true

3. 检查数据库初始化
   └─> 运行 await cognee.add()
   └─> 运行 await cognee.setup()

4. 检查图谱是否构建
   └─> 运行 await cognee.cognify()
   └─> 检查图谱是否为空

5. 检查权限配置
   └─> 确认 ENABLE_BACKEND_ACCESS_CONTROL
   └─> 检查用户数据集权限

6. 查看详细日志
   └─> 设置 ENV="debug"
   └─> 查看 ./logs/ 中的日志文件
```

按照这个流程，90% 的启动问题都可以解决！
